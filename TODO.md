# ShadowNote · 待办 / Roadmap

记录接下来想做的几件事。完成一项就把 `[ ]` 改成 `[x]`。

## 1. 文档：用通义听悟获取逐字稿

写一段简短指引（放进 README，或单独建 `docs/transcript.md`），让用户知道怎么拿到带时间戳的逐字稿。

- [ ] 基本流程：上传课程视频/音频 → 等待自动转写 → 导出逐字稿（带时间戳）
- [ ] 导出格式建议 `.docx` 或 `.txt`；工作台能直接导入 `.docx`（用 mammoth 解析），也支持 `.srt` / `.vtt` / 粘贴
- [ ] 提醒：ASR 对英文术语容易听错（如 Bayes → basement、law of total probability → love total probability），用工作台的「逐字稿清理」一步纠正
- [ ] 确认通义听悟**当前**的导出按钮/步骤名称（界面可能已更新，写文档前再核对一遍）
- [ ] 可顺带列一两个备选转写工具

## 2. 自制视频注释（交互式打点）

不依赖逐字稿、不依赖 LLM，边看视频边手动做笔记。

- [ ] 播放到某处 → 一键「打点」记录当前时间戳（取 `video.currentTime`）
- [ ] 给这个点写标题 / 重点 / 笔记，边看边加、可编辑
- [ ] 时间戳可拖动微调
- [ ] 导出成 `notes.txt` / `notes.json`，和现有格式打通
- 价值：给"手动派"一条不花 token、不用转写也能造笔记的路

## 3. 字幕版

把笔记/逐字稿当字幕叠在视频上。

- [ ] 用（清理后的）逐字稿生成字幕，显示在视频底部
- [ ] 导出 `SRT` / `VTT`
- [ ] 全局时间偏移（±秒），用来对齐"缝别人的视频"；进一步可做两点锚定
- 备注：无字幕版已完成，字幕作为独立迭代叠加上去
